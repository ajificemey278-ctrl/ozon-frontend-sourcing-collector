# 平台流程

## Ozon

适合采集：竞品标题、价格、币种、评分、评价数、到货时效、图片结构、卖点文案。

推荐流程：

1. 用 `build-url --platform ozon --keyword "关键词"` 生成搜索链接。
2. 在浏览器里打开，确认地区、币种和到货时效显示正常。
3. 复制搜索结果可见文本，或打开具体商品页后保存 HTML。
4. 用 `parse-text` 或 `parse-html` 输出 JSON。
5. 只把 Ozon 属性作为竞品基准，不当作国内货源真实属性。

## 1688

适合采集：工厂/批发货源标题、价格、起批量、销量、评价、店铺、规格、发货和属性线索。

关键规则：1688 搜索关键词必须使用 GB18030 编码，否则中文容易乱码。

推荐流程：

1. 用 `build-url --platform 1688 --keyword "中文关键词"` 生成链接。
2. 在浏览器里打开链接，不用程序高频请求搜索页。
3. 看到验证码、滑块、登录或安全验证时，人工处理；处理不了就停止。
4. 复制可见文本或保存详情页 HTML 后解析。
5. 优先保留单件可买、有销量评价、价格运费清楚、属性证据清楚的货源。

## 淘宝/天猫

适合采集：零售货源标题、价格、销量/评价、店铺、SKU、运费和买家可见属性。

推荐流程：

1. 用 `build-url --platform taobao --keyword "中文关键词"` 生成链接。
2. 用用户自己的浏览器正常打开。
3. 如果要求登录/滑块/验证码，人工处理；工具不绕过。
4. 解析用户复制的可见文本或保存的 HTML。

## 义乌购

适合采集：义乌批发货源、店铺、价格、起订量、供货信息。

推荐流程：

1. 用 `build-url --platform yiwugo --keyword "中文关键词"` 生成链接。
2. 打开并筛选能单买或小批量采购的商品。
3. 复制可见文本或保存 HTML 解析。

## 输出给后续分析的最低字段

- 平台和来源链接。
- 标题。
- 价格文本、币种、价格数值。
- 销量/评价/评分。
- 店铺名。
- 到货/发货/运费信息。
- 商品规格、重量、尺寸、数量、功能属性的证据文本。
- 是否触发验证，以及触发原因。
