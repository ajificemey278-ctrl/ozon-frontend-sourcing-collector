# 安全采集边界

本项目只用于采集公开前端可见信息，服务 Ozon 选品、竞品和国内货源分析。

## 允许

- 用户正常打开网页后，保存 HTML 或复制可见文本，再用工具解析。
- 单个公开 URL 的低频请求，用于确认页面是否可公开访问。
- 生成 Ozon、1688、淘宝/天猫、义乌购搜索链接。
- 从可见内容提取标题、价格、币种、评分、评价、销量、店铺、到货/发货、规格线索和图片线索。

## 不允许

- 绕过验证码、滑块、登录墙、风控页。
- 自动登录、保存账号密码、保存 Cookie、导入浏览器用户数据。
- 使用代理池、IP 轮换、批量并发、高频重试。
- 伪造真实用户行为绕开平台限制。
- 把竞品页面的属性直接写成国内货源真实属性。

## 遇到验证怎么处理

1. 工具输出 `verification_required` 后停止。
2. 用户可以在浏览器里人工处理验证。
3. 验证后重新保存页面或复制可见文本再解析。
4. 如果仍然触发风控，停止该平台或该链接，不继续重试。

## 风险控制默认值

- 每次运行同一 host 只允许 1 次网络请求。
- `fetch-url` 必须显式加 `--allow-network`。
- 默认等待 8 秒再请求。
- 403、429、登录、验证码、风控关键词都会停止。
