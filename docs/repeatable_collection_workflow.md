# 可重复采集流程

目标：每次都用同一套低风险动作，把 Ozon 竞品和国内货源前端信息整理成同一份 JSON，而不是临时截图、临时口径。

## 一次任务的标准步骤

1. 明确关键词和平台。
2. 用 `build-url` 生成平台搜索 URL。
3. 人工打开 URL，确认页面不是验证码、登录墙、风控页。
4. 采集页面可见文本或保存 HTML。
5. 用 `parse-text` 或 `parse-html` 输出 JSON。
6. 如果是单个公开详情页，也可以用 `fetch-url --allow-network` 低频采一次。
7. 后续分析只读 JSON，不直接猜页面内容。

## 平台顺序建议

1. Ozon：先看竞品价格、评价、到货时效、图片打法、卖点和差评痛点。
2. 1688：优先找工厂/批发货源，注意 GB18030 搜索编码。
3. 淘宝/天猫：补零售现货、评价和买家图线索。
4. 义乌购：补义乌供货源和批发价线索。

## 停止条件

出现以下任一情况必须停止该链接，不继续请求：

- 验证码、滑块、登录墙。
- HTTP 401/403/407/429。
- 页面提示访问受限、风控、安全验证。
- 同一 host 已经请求过 1 次。
- 页面需要绕过限制才能看到内容。

## 输出判断

`status = ok` 才能进入后续分析。

`status = verification_required` 需要人工处理验证后重新提供 HTML/文本。

`status = network_error` 说明网络或远端异常，不作为商品结论。

## 质量标准

- 价格必须保留 `price_text`、`currency`、`price_value`。
- Ozon 页面显示 `¥` 就是 CNY，显示 `₽` 就是 RUB，不混用。
- 国内货源属性必须来自国内货源页面证据。
- Ozon 竞品属性只能做对标，不能直接当作货源真实属性。
- 搜索页卡片是低置信度，详情页/可见参数/评价证据置信度更高。
